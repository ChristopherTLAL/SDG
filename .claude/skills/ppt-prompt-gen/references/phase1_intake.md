# Phase 1 — 意图对齐清单

## 目标

收集 9 项关键信息，全部 explicit 落地后才能进 Phase 2。但**不要一次性把 9 个问题甩给用户**——这是糟糕的体验。正确做法：

1. 先把用户已经给的材料 / 上下文消化一遍
2. 凡能从材料里推断出来的，先推断、先填、再让用户确认 "我理解的是 X，对吗？"
3. 真正无法推断的，**合并成 2-3 轮提问**（一轮 3-4 个相关问题），不要一问一答 9 次

## 9 项清单

每项都有"如何推断 / 怎么问"指引。

### 1. 主标题 + 副标题

- **从材料推断**：看材料标题 / 文件名 / 学生姓名 + 主题。例 "{学生} 选校汇报"、"{产品} 路演"。
- **问法**："我先暂定主标题为 'X'，副标题为 'Y'，你看？"
- **要点**：标题不要太长（中文 ≤ 12 字最佳）；副标题给受众一个"为什么我应该听"的钩子

### 2. 受众 + 场景

- **从材料推断**：留学规划材料 → 客户家长 + 学生；公司 deck → 内部 / 投资人 / 客户；学术报告 → 同行
- **问法**："这份 deck 是讲给谁听？什么场景？" 同时给 2-3 个候选受众让用户选
- **影响**：受众决定行话密度、tone、叙事弧、字号

### 3. 演讲时长

- **从材料推断**：通常推不出来，必须问
- **问法**："大概讲多长时间？" 给区间："5 分钟（电梯）/ 15 分钟（短汇报）/ 30 分钟（标准）/ 60+ 分钟（培训）"
- **影响**：时长 → 页数（见 phase2_outline.md 的页数公式）

### 4. 核心目标

- **从材料推断**：从材料的"结论"、"建议"、"行动项"段落能推。如果材料是中性叙述（如研究综述），目标可能是"传授知识"
- **问法**："讲完之后你希望受众做什么 / 记住什么？" 给候选：
  - 决策（同意 / 批准某事）
  - 行动（开始 / 改变某行为）
  - 认知（理解某概念 / 了解某情况）
  - 说服（接受某主张 / 投票支持）
  - 鼓舞（产生情感共鸣）
- **影响**：目标决定 narrative arc 选型 + summary 页的"call to action"措辞

### 5. 必含数据点 / 论据

- **从材料推断**：列出材料里的关键数字 / 表格 / 引用 / 案例，标注"我觉得这些是必须出现的"，让用户增删
- **问法**："这些点是必须出现在 deck 里的（列出），还有遗漏吗？哪些可以省略？"
- **要点**：宁多勿少；Phase 2 outline 时再决定哪些进主线、哪些进附录

### 6. 叙事框架建议

- **从材料推断**：看材料是"问题驱动"（开篇就有 pain point）还是"机会驱动"（开篇是 vision），分别对应不同 arc
- **问法**：从 `phase2_outline.md` 的 arc 库里推荐 1-2 个候选，简短解释适用场景，让用户选 / 或说 "自由"（你来定）
- **要点**：用户没经验时不要让他从 7 个 arc 里选，直接推荐 1 个 + 一句话解释

### 7. Tone & Style

- **候选**：学术严谨 / 商业咨询 / 销售热情 / 教育温和 / 极客科技 / 故事化叙事
- **从场景推断**：留学规划 → 教育温和；投资人路演 → 商业咨询 / 销售热情；学术 → 学术严谨
- **问法**："tone 我准备用 'X'，比如 ABC（举一两个例子）。OK？"
- **影响**：影响标题 / body 文案的措辞选择（Phase 3 文案生成时用）

### 8. 语言

- **默认**：内容中文 + 必要 jargon 英文（中英混排）
- **问法**：仅在有歧义时问。如果受众明显是英文听众（如海外投资人 / 国际学校面试），主动问"做英文 deck 吗"
- **影响**：决定每页 prompt 里 `title_en` 是否生成、body 文字语言

### 9. 风格 pack 选择

- **从触发语推断**：用户说"瑞士风" / "swiss" → `swiss-international`；"杂志风" / "editorial" → `editorial-magazine`；"科技 / 暗色 / 路演" → `dark-techno`；"咨询 / 优雅 / 紫色" → `soft-dusty-purple`
- **从场景推断**：客户汇报 + 留学场景 → 默认推荐 soft-dusty-purple；学术 / 数据密集 → swiss；品牌内容 / 长图文 → editorial；TMT 公司 / 投资人 → dark-techno
- **问法**：列 4 个风格 + 一句话特征，让用户选。或者用户说"你推荐"，按场景推 1 个 + 简短理由
- **要点**：风格一旦定下来 → 整套 deck 必须用这一个 pack，不要混搭

## 退出 Phase 1 前的最终回执

收集完 9 项后，用一个清晰的 confirmation 块给用户看：

```
意图已对齐：

  主标题：王涵润 英国本科申请规划
  副标题：化工方向 · 2026 fall 入学
  受众：学生本人 + 家长（中性偏教育角色）
  场景：30 分钟一对一沟通
  时长：30 分钟（推荐 12-15 页，详 Phase 2）
  目标：让家长清晰理解时间线 + 关键决策点
  必含：雅思节奏（6 月首考）、MBTI INFJ + Holland ACR、3 个赛道（化工/医疗器械/医药研发）
  叙事：问题-分析-方案-收益
  Tone：教育温和 + 一点商业理性
  语言：中文为主，雅思 / IELTS / TMUA 等保留英文
  风格：soft-dusty-purple

确认无误请回复 "继续"，要改的话直接说。
```

用户回 "继续 / OK / 可以 / 行" → 进 Phase 2。
用户改某项 → 改后再确认，再 ask 是否继续。
不要假定用户回 "嗯" 就是确认——必须明确肯定才进下一步。

## 常见坑

- **不要假设受众 = 用户**：用户是王世杰（顾问），受众可能是家长 / 学生 / 投资人 / 老板
- **不要先入为主推荐风格**：用户没说风格时，要么推荐 1 个 + 让他确认，要么列 4 个让他选；不要默认 soft-dusty-purple
- **必含数据点要"具体"**：不是"包括雅思情况"，而是"包括 6 月首考目标 + 7-8 月集训 + 8 月二战 6.0+ 这条节奏"。具体到能直接写进 prompt
