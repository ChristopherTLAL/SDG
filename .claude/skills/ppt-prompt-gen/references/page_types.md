# 页面类型 Taxonomy

每个 deck 由若干种页型组成。不同页型的信息密度、视觉重心、可视化策略完全不同——封面页谈不上 "2-3 种可视化"，数据页则相反。Phase 2 outline 必须给每页打上 `page_type` 标签，Phase 3 生成 prompt 时按页型差异化处理。

## 9 种核心页型

### 1. `cover` — 封面页

- **作用**：建立第一印象，定基调，传达核心命题
- **核心元素**：主标题（必）、副标题（可选）、tagline（可选）、汇报人 / 日期 / 受众 / logo
- **信息密度**：极低，留白率 ≥ 50%
- **可视化**：通常 0 个图表；可有 1 个抽象视觉符号（几何形状 / 摄影底图 / 大字 typography）
- **禁忌**：罗列议程（那是目录页）、堆砌数据（那是数据页）
- **prompt 重点**：typography 层级（H1 极大）、视觉锚点位置（左对齐 / 居中 / 右下）、背景处理（纯色 / 渐变 / 纹理）

### 2. `toc` — 目录 / Agenda

- **作用**：建立心理预期，让受众知道接下来 N 分钟会经历什么
- **核心元素**：章节编号 + 章节标题 + （可选）每章一句话简介 / 页码范围
- **信息密度**：低，留白率 30-40%
- **可视化**：左侧导航条 / 圆点序号 / 简单 timeline 形态；不要复杂图表
- **禁忌**：超过 7 个章节（认知超载）、章节标题超过 1 行
- **prompt 重点**：序号 typography 是视觉锚点，章节间距均匀

### 3. `section_divider` — 章节页 / 转场

- **作用**：在长 deck 里给受众"翻页喘息"，标记叙事节奏
- **核心元素**：章节编号（大字）+ 章节标题 + （可选）章节摘要 / 进度条
- **信息密度**：极低，留白率 ≥ 50%
- **可视化**：通常 0；可有装饰性几何 / 进度条
- **禁忌**：当成内容页用（堆数据）、和封面页风格混淆
- **prompt 重点**：和封面页要有视觉区别（通常背景色翻转 / 加章节编号）

### 4. `content` — 内容页（主力）

- **作用**：deck 的主体，承载 80% 的论证 / 分析 / 信息
- **核心元素**：H2 区块标题 + 2-4 个并列模块（每模块可含小标题 + body + 图 / icon）
- **信息密度**：中-高，留白率 15-25%（视风格而定，soft-dusty-purple 偏密、editorial 偏疏）
- **可视化**：每页 1-3 个图表 / icon 组合；常用 chevron process、harvey ball、bar chart、stat block
- **禁忌**：一页讲两个 H2 主题（拆成两页）、文字段落超过 3 行（改成 bullet）
- **prompt 重点**：模块分区清晰（用色块 / 分割线 / grid）、视觉锚点指引读者顺序

### 5. `data` — 数据页

- **作用**：聚焦呈现 1-2 个关键数据 / 图表，让数字说话
- **核心元素**：H2 标题 + 1 个主图（占页面 50-70%）+ 关键洞察一句话 + 数据来源
- **信息密度**：图表本身高，但页面整体留白 25-35%
- **可视化**：1 个主图表（绝不简单饼图）+ 1-2 个辅助 stat block / sparkline
- **禁忌**：5+ 图表挤一页（拆成多页或换 dashboard 类页型）、没有"so what" 洞察
- **prompt 重点**：高亮色仅给核心数据点，其他全灰；图表标题 / 轴标 / 图例 typography 严格遵循风格

### 6. `comparison` — 对比页

- **作用**：A vs B / 多选项并列 / before-after / 我们 vs 对手
- **核心元素**：左右两栏（或多栏并列），每栏对称结构，明显的"对比维度"行
- **信息密度**：中，留白率 20-30%
- **可视化**：butterfly chart、对称 bar、slope graph、harvey ball 矩阵、icon 对照表
- **禁忌**：栏与栏结构不对称（破坏对比）、栏数 > 4（改成表格页）
- **prompt 重点**：栏间分割线 / 色块、"赢家"用强调色高亮、维度行用浅底色统一

### 7. `process` — 流程 / 时间线

- **作用**：表达步骤、阶段、时序、因果链
- **核心元素**：步骤节点（3-7 个）+ 每节点小标题 / 描述 + 流向箭头 / 连线
- **信息密度**：中，留白率 25-30%
- **可视化**：chevron、gantt、timeline、circular process、swimlane、value stream
- **禁忌**：步骤超过 7 个（拆成 phase A 和 phase B 两页）、流向不清
- **prompt 重点**：箭头 / 连线用浅灰，节点高亮当前 / 关键步骤

### 8. `quote` — 金句 / 引用页

- **作用**：节奏调节，给受众停顿；强调一个核心观点 / 客户证言 / 专家引用
- **核心元素**：大字引言（H1 体量）+ 引用来源（小字）+ （可选）小图像 / 装饰
- **信息密度**：极低，留白率 ≥ 60%
- **可视化**：0 个图表；可有大引号符号 / 头像 / 抽象装饰
- **禁忌**：放数据图表、文字超过 2 行
- **prompt 重点**：typography 是唯一视觉元素，引号字符要大且艺术化

### 9. `summary` — 总结 / Takeaways

- **作用**：deck 结尾的"带走"——3-5 个 takeaway / 行动号召 / 下一步
- **核心元素**：H1 "Key Takeaways" / "下一步" + 3-5 条编号要点
- **信息密度**：低-中，留白率 25-35%
- **可视化**：编号序列、icon list、harvey ball 完成度；不要复杂图表
- **禁忌**：takeaway 超过 5 条（受众记不住）、和内容页混淆
- **prompt 重点**：编号 / icon 是视觉锚点，每条 takeaway 一行字内

## 可选页型（按需）

- `appendix` — 附录页，仅作背书备查，不进主流程，密度自由
- `qa` — Q&A 页，类似 quote 页但带 placeholder "Questions?"
- `thanks` — 感谢页，封面页同构反向版

## 每个 deck 的页型节奏（建议）

不是死规则，但好 deck 通常有这个节奏感：

```
cover → toc → section_divider → 2-4 × (content | data | comparison | process) 
→ section_divider → 2-4 × (...) → ... → summary → thanks
```

**节奏检查**：连续 4+ 页同一页型（比如 6 个 content 连在一起）= 受众疲劳，插入 quote / section_divider / data 切换节奏。Phase 2 outline 要主动检查这点。
