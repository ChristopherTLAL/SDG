# Phase 2 — Outline 规划

## 目标

输出 3 样东西给用户看 → 等用户改 / 确认 → 才能进 Phase 3。

1. **总页数推荐**（带依据）
2. **Narrative arc**（从下方库里选 1 个）
3. **逐页 outline 表**（页号 / 页型 / 标题 / 核心信息 / 候选图表）

## Part 1 — 总页数推荐公式

页数不是凭感觉，按以下经验公式给出**区间**让用户选：

| 场景 | 时长 | 推荐页数 | 备注 |
|:--|:--|:--|:--|
| 电梯 pitch | 3-5 min | 3-5 页 | 每页 1 分钟，密度极高 |
| 短汇报 | 10-15 min | 8-12 页 | 每页 1-1.5 分钟，主力页型 + 1-2 个 data 页 |
| 标准客户汇报 | 25-35 min | 15-22 页 | 节奏感强，每 4-5 页插一个 section divider / quote |
| 内训 / Workshop | 45-60 min | 25-35 页 | 多 process / data 页，可加附录 |
| 深度技术讲座 | 60-90 min | 30-45 页 | 学术 / 极客向，density 高 |
| 路演 Demo Day | 8-10 min | 10-12 页 | 经典 pitch deck 范式 |

### 调节因子

基础区间 ± 以下因子：

- **受众偏专业** → +20% 页数（可以承受更高 density 但每页更深）
- **受众偏外行** → -20% 页数（每页讲透一个点，多 quote / 视觉化）
- **必含数据点 > 10 个** → +3-5 页
- **风格是 editorial-magazine**（每页留白大）→ +20% 页数
- **风格是 swiss-international**（密度高）→ -10% 页数
- **首次接触受众**（需要建立信任）→ +1 cover page + 1 about 页

输出给用户："我建议 X 页（区间 X-Y），理由：时长 N 分钟 + 受众 + 必含点数。如果想更精简 / 更详尽告诉我。"

## Part 2 — Narrative Arc 库

每个 arc 适合不同的沟通目标。Phase 1 已经收集了"核心目标"，按目标推荐：

### Arc 1：SCQA（Situation-Complication-Question-Answer）

- **结构**：现状 → 矛盾 / 痛点 → 关键问题 → 你的答案
- **适用**：决策导向、咨询汇报、说服性场景
- **典型页面分布**（15 页）：
  - 封面 1 / Situation 3 / Complication 3 / Question 1 / Answer 5 / Summary 1 / Q&A 1

### Arc 2：问题-分析-方案-收益（Problem-Analysis-Solution-Benefit）

- **结构**：问题陈述 → 深度分析 → 提出方案 → 预期收益
- **适用**：项目提案、内部立项、客户问题诊断 + 解决
- **典型页面分布**（12 页）：
  - 封面 1 / TOC 1 / Problem 2 / Analysis 4 / Solution 3 / Benefit 1 / Closing 0
- **王世杰留学规划高频用**：问题（学生现状卡点）→ 分析（评估 / 测评）→ 方案（时间线 + 学校 + 文书）→ 收益（预期 offer）

### Arc 3：Pyramid Principle（金字塔）

- **结构**：主张（top）→ 3 个支撑论据 → 每论据下的细节论证
- **适用**：高管汇报（时间紧，结论先行）、需要被 challenge 的提案
- **典型页面分布**（10 页）：
  - 封面 1 / 主张 1 / 论据 1+ 细节 2-3 / 论据 2 + 细节 2-3 / 论据 3 + 细节 2-3 / Closing 1

### Arc 4：AIDA（Attention-Interest-Desire-Action）

- **结构**：抓注意力 → 激发兴趣 → 引发渴望 → 推动行动
- **适用**：销售 deck、产品发布、营销提案
- **典型页面分布**（10 页）：
  - 封面 / 吸睛事实 1-2 / 产品 / 服务介绍 3 / 价值 / 收益 2-3 / CTA 1

### Arc 5：Why-What-How（Simon Sinek 黄金圈）

- **结构**：为什么这件事重要 → 是什么 → 怎么做
- **适用**：愿景驱动、品牌宣讲、文化布道
- **典型页面分布**（12 页）：
  - 封面 / Why 3-4 / What 3 / How 4 / Closing 1
- **要点**：Why 段是情感钩子，多用 quote / big number / 视觉锚点

### Arc 6：Past-Present-Future（时序）

- **结构**：过去（背景 / 起点）→ 现在（成果 / 状态）→ 未来（计划 / 愿景）
- **适用**：年度回顾、个人故事、公司发展史 → 战略展望
- **典型页面分布**（12 页）：
  - 封面 / Past 3 / Present 4 / Future 4 / Closing

### Arc 7：Compare-Contrast（并列对比）

- **结构**：A 方案 / B 方案 / C 方案 → 各自维度对比 → 推荐
- **适用**：方案选型、多产品对比、多学校对比（**留学场景**）
- **典型页面分布**（10 页）：
  - 封面 / 评估维度 1 / A 介绍 2 / B 介绍 2 / 对比矩阵 1 / 推荐 1 / Q&A

### Arc 8：Hero's Journey（英雄之旅，简化版）

- **结构**：日常 → 召唤 → 挑战 → 转折 → 凯旋
- **适用**：案例研究 / 客户证言 / 创业故事
- **典型页面分布**（10 页）：
  - 封面 / 主角介绍 1 / 困境 2 / 转折点 2 / 解决方案 2 / 结果 2 / Lesson 1

### Arc 9：自由 / 自定义

- 当上述都不贴合时，让 AI 根据材料自己设计 arc 结构 → 在 outline 表里明确标出每页的 narrative_position
- **要点**：自定义 arc 必须给用户一个"为什么这样排"的简短解释，让用户能 challenge

### 怎么推荐 arc

| 用户给的核心目标 | 推荐 arc（优先） |
|:--|:--|
| 决策 | SCQA / Pyramid |
| 行动 | AIDA / Why-What-How |
| 认知 | Past-Present-Future / Compare-Contrast |
| 说服 | SCQA / Pyramid |
| 鼓舞 | Why-What-How / Hero's Journey |
| 留学规划方案 | 问题-分析-方案-收益（特化） |
| 选校汇报 | Compare-Contrast |

## Part 3 — Outline 表格式

给用户的最终 outline 表必须是 markdown 表格，**包含 narrative position 列**——这是叙事连贯性的关键锚。

```markdown
## Outline — 共 12 页，arc: 问题-分析-方案-收益

| # | 页型 | Narrative | 标题 | 核心信息 | 候选图表 |
|:-:|:--|:--|:--|:--|:--|
| 1 | cover | — | 王涵润 英国本科申请规划 | 化工方向 · 2026 fall 入学 | 抽象几何 |
| 2 | toc | — | 议程 | 4 大议题 30 分钟 | 4-step 导航条 |
| 3 | section_divider | problem-intro | Part 1：当前状况 | — | 大字"01"+ 进度条 |
| 4 | content | problem-1/2 | 学术现状 | A-Level 3 科预估 A*AA，雅思未考 | Harvey Balls × 3（A-Level / 雅思 / 标化）|
| 5 | data | problem-2/2 | 目标学校录取门槛 | UCL/IC/Bristol 化工的硬指标差距 | Slope Graph（学生 vs 目标）|
| 6 | section_divider | analysis-intro | Part 2：诊断分析 | — | 大字"02" |
| 7 | content | analysis-1/3 | 性格 + 兴趣评估 | MBTI INFJ + Holland ACR | 2×2 Matrix |
| 8 | content | analysis-2/3 | 3 个赛道适配度 | 化工 / 医疗器械 / 医药研发 | Radar Chart |
| 9 | content | analysis-3/3 | 关键卡点 | 雅思 + 文书 + 入学考试 | Pyramid + Harvey Balls |
| 10 | section_divider | solution-intro | Part 3：执行方案 | — | 大字"03" |
| 11 | process | solution-1/2 | 12 个月时间线 | 6 月首考 / 7-8 集训 / 10 月 UCAS | Gantt Chart |
| 12 | content | solution-2/2 | 选校建议（5 所） | 冲刺 1 / 主申 3 / 保底 1 | Stat Cards × 5 |
| 13 | summary | benefit | 预期收益 | 2 个 offer 概率、提升路径 | Big Number + Funnel |
| 14 | thanks | — | 谢谢 | — | 简洁 |

**节奏检查**：✅ 每 3-4 页插了 section_divider；✅ 连续 content 不超过 3 页；✅ 包含 1 个 process、1 个 data、1 个 summary。

是否要改？比如增删页 / 换图表 / 调叙事顺序？告诉我，我改完再确认。
```

## Part 4 — 用户改 outline 的常见 pattern

| 用户说 | 怎么改 |
|:--|:--|
| "页太多了，砍 X 页" | 优先砍 section_divider / 合并相邻 content / 砍 appendix |
| "X 那一页加点 Y" | 在该页 core_message 里加 Y；如果信息量爆掉就拆成 2 页 |
| "换个 arc 试试" | 重新生成 outline 表（成本不高，直接重做）|
| "图表别用 X 用 Y" | 改 chart_types 列；其他不动 |
| "封面用纯文字" | cover 页的 chart_types 改为 `["typography only"]` |
| "加一页 about me / 团队" | 在 cover 后插入 1 页 content 类型，narrative 标 `intro` |

## 退出 Phase 2 前

用户说 "可以 / 这样 / 继续 / 开始生成" → 进 Phase 3。
用户没明确确认 → 再问 "outline 这样可以吗？没问题我开始生成 prompts"。

**不要 outline 一改就 silently 进 Phase 3**——每次改完都要等用户再确认一次。
