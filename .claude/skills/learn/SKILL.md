# 博学伴读

给王世杰自己的深度学习工具。遇到任何一个元素想搞懂——一个词、一个人、一款茶、一部电影、一个概念——产出一篇平实的深挖随笔：从最强的一条线讲到底，读完得到一个完整的认识。

## 初心：一个元素，一条线，一个完整的认识

深度 ≠ 信息量大。最容易犯的错是**发散**：把所有冷知识堆上来，读者什么也学不会。正确做法是从素材里挑出**最强的一条主线**，讲透，不为它服务的料一律砍掉。读者读到底，得到一个记得住的认识，而不是一堆碎片。

## 怎么跑：深搜 → 写稿 → 审核重写（三步，第三步必做）

1. **深搜，走 dynamic workflow（`workflow.mjs`）。** fan-out：多个 agent 各查一个角度（源 / 工艺 / 名 / 传播 / 谱系 / 跨域连接），深挖具名的人事物、确切年代，按 实 / 争 / 俗 核实，汇成一份厚实、分档的连接素材档 + 几条「候选主线」。只搜不写。
   - 实操：触发后用 **Workflow 工具**跑 `learn/workflow.mjs`，传 `args = {topic, note}`，拿回 `dossier`。`args` 偶尔不稳，拿不到就把对象写进脚本顶部 `TOPIC_INLINE` 再 `Workflow({scriptPath})` 重跑。
2. **写稿，主窗口 inline。** 完整读 `references/voice.md`，从 dossier 里**挑一条主线**，把不为它服务的料砍掉，按那套平实的声音写成一版草稿。
3. **审核重写，起一个编辑子 agent（必做，别拿草稿直接交付）。** 单遍写压不住模型往文学性跑的本能，所以草稿写完**必须**再起一个子 agent 当编辑：让它完整读 `references/voice.md`（尤其「审核清单」那节），把草稿逐条过一遍，改掉所有违反项——文学性、繁体字、半角标点、「不是 X 是 Y」、破折号、诗意结尾、没解释的术语。**只许改语气和这些机械问题，不许动事实 / 数据 / 引用 / 主线。** 拿它重写的版本交付。
   - 交付前再跑个机械兜底，揪漏网的繁体字和中文半角逗号（成稿存 `/tmp/a.md` 后）：
     ```bash
     python3 -c "import re;t=open('/tmp/a.md').read();print('繁体:',{c for c in t if c in '後來裡裏麼與這們個時對為觀點關發會說書當還萬廣國學體邊樣寫讀東車門無黨醬'} or '无');print('中文半角逗号:',re.findall(r'[一-鿿],|,[一-鿿]',t)[:8] or '无')"
     ```
     有就当场改掉再交。

## 六透镜（深搜阶段把角度铺全用，不是写稿清单）

源（从哪来）/ 流（怎么变）/ 对（跟谁像、差在哪）/ 网（谁影响它、它影响谁）/ 用（怎么用、常错在哪）/ Aha（反直觉的跨域连接）。搜资料时用它把角度铺全；**写稿时只挑服务那条主线的几根，不是六根都写。**

## 反编造：实 / 争 / 俗（命门）

起源、年代、归属、「A 影响 B」是幻觉重灾区，而且最好听的连接往往是假的。深搜阶段每条断言标三档：实（学界 / 权威共识）、争（有争议 / 一般认为）、俗（民间附会）。写稿阶段：实的直接讲，争的说清存疑，俗的不提或点破。词源走 etymonline / OED 思路核。宁可写「这条存疑」，绝不把好听的假话讲成真的。

## 声音

全在 [`references/voice.md`](references/voice.md)，写稿阶段必须完整读它。一句话：**平铺直叙、零表演，一条线、一个完整的认识。**
