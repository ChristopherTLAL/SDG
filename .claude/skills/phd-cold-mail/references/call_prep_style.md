# 面谈准备包 写作规范（王世杰 house-style）

> 准备包是**中文分析 + 英文问题**的交付物。分析部分的写作标准对齐 ps_writer/style_guide.md，但落在中文上。每份准备包写完，按这张表自查（多数能 grep）。

## 硬规则（违反就改）

1. **无破折号。** 不用 `——`、`—`、`–`。要停顿用逗号或句号，或直接拆成两句。数值范围用「到」或「至」（0.1 到 20 Gy），不用 `0.1–20`。

2. **引号少用，且不装饰。**
   - 能转述就转述，不要给术语、概念加引号当强调。
   - 只在**确实引用对方原话 / 论文标题**时用，中文用 `「」`。
   - 禁止用引号做"反讽 / 加重语气"（如 他的"主业"、所谓的"考核"）。

3. **禁 AI 对仗句式。** 最常见三种，看到就重写：
   - `不是 X，而是 Y` / `不只是 X，更是 Y`
   - `与其说 X，不如说 Y`
   - `X 不是终点，Y 才是`
   改成直接陈述结论：先说 Y 是什么，需要时再补一句 X 的澄清。

4. **去 conceptual metaphor / 成语化修辞 / 夸张情绪词。**
   - 不用 `火力全开`、`一针见血`、`阴魂不散`、`暴涨`、`碾压`、`闭环` 这类。
   - 判据一条：这句要靠语感才能一遍读懂吗，要就换字面。
   - 英文里同理：去 delve / testament / profound / groundbreaking。

5. **禁水词过渡。** 中文不用 `值得注意的是`、`换句话说`、`不难发现`、`综上所述`；英文不用 Furthermore / Moreover / In addition。直接接下一句。

6. **大白话，自信精准。** 像有经验的导师当面讲给得意学生听，直接、支持、关键处一句说透，但不绕金句。

7. **忠实。** 只用素材里真实出现的经历 / 数字 / 方法。研究部分的每个论文断言可追溯到来源。不臆造。

8. **英文问题保留英文**（call 里就这么问 / 答），中文只点一句意图。

9. **结尾给 APA7 reference list，且每个 DOI 必须 CrossRef 核验。** 研究部分引到的论文，结尾统一列 APA7 格式（页码范围用连字符，不用 en-dash）。来源页面（官网 / Scholar / GitHub）单列。
   - **🚨 第一关 = 作者归属：每一篇你说成是这位老师的论文，CrossRef 的作者列表里必须真的有这位老师本人。** 这是冷邮件那边漏掉、害我们把一篇真论文安错人的坑：一篇主题完美但出自**别的课题组**的真实论文。准备包是照着我们写的冷邮件做的，而冷邮件本身可能就把论文安错了人（真实事故：把一篇 2023 *Nature Communications* 的 human-in-the-loop ML 肽水凝胶论文算到 **Saiani** 头上，实际作者是**西湖大学**某组，Saiani 根本不在署名里）。所以**不要直接信冷邮件的署名**，每篇都自己开 `https://api.crossref.org/works/{DOI}` 看 `author` 数组里有没有这位老师的姓。没有就从准备包里删掉，哪怕它主题最贴。这和冷邮件起草端的 CrossRef gate 是**同一把锁，两边都要上**。
   - **第二关 = DOI / 元数据对齐：** 用 `https://api.crossref.org/works/{DOI}` 逐个核对年份 / 期刊 / 卷期 / 页码。LLM 经常幻觉 DOI 数字（Graham 2019 那篇就把 `5056170` 写成 `5056665`，404）。404 或元数据不符就反查 `api.crossref.org/works?query.bibliographic=标题+作者` 拿正确 DOI。论文存在但 DOI 错，是最隐蔽的坑。
   - 可直接跑 `scripts/verify_citations.py`（同一把 CrossRef 作者归属 gate）批量核验，再人工补查无 DOI 的引用。

10. **不暴露内部工具 / 局限。** 别写 `因 repository 屏蔽自动抓取`、`搜索没返回`、`子代理` 这类内部 plumbing。学生只看结论。要提示时效就直接说「call 前去 Scholar 看有没有更新」，不带借口。

11. **不要 signposting。** 砍掉 `· 深度解读（重点）`、`（call 前务必读）`、`重点看第二部分` 这类告诉读者怎么用的元话。内容本身够清楚就不用标。小标题只写它是什么（`Graham 在做什么`，不是 `Graham 在做什么 · 深度解读`）。

12. **conceptual metaphor 扩展清单（中文也算）。** 除 §4，再补：`最稳的打法 / 打法`（战斗游戏比喻，换「建议这样做」）、`硬编`（换「现编一个答案」）、`接你的点`（换「对你的意义」）、`抓手 / 落点 / 闭环 / 阴魂不散`。

13. **三种格式 + 它们之间的间距（侧边栏不重叠）。** 准备包用三种排版，缺一不可，别只用「正文 + callout」（最容易漏掉中间那档引用块）：
    - **正文**：主线解读，平铺直叙。
    - **`>` 引用块**（顶部 `对象 / 场景 / 用法` 头、`批注：`、`术语注：` 等旁注）：渲染成带左侧边栏的引用框，放旁注、术语、称呼提醒。
    - **`> [!INSIGHT]` callout**：最关键的战略判断（接 RP、别踩的坑），渲染成框。
    **间距铁律**：两个 `>` 块（引用或 callout 都算）之间**必须有真正的内容隔开**，否则在 Obsidian 侧边栏 / 预览里两个框会挤在一起重叠。
    - 「真正的内容」= 一段正文 / 一个列表 / 一条 `---`。**只隔一个空行不够，只隔一个标题也不够**（标题太短，两个框还是贴着）。
    - 推论：**别让一个小节以 `>` 块结尾、下一个小节又以 `>` 块开头**（它们会夹着标题贴在一起）。每个 `>` 块前后都要有正文句子托着。
    - 顶部 `对象 / 场景 / 用法` 引用块是唯一例外（紧接 `---`，本身是文档头）。一段 section 的引导说明用正文（或 `*斜体*`），别用 `>`。
    - 自查脚本：扫所有 `>` 组，若相邻两组之间只有空行和 / 或标题（无正文 / 列表 / `---`）→ 改。

14. **🎯 connection-making 是准备包的灵魂，不是装饰。** 解读老师的研究**不是为了考学生背论文**，是为了让学生当场把老师的工作接到**我们自己的 RP（研究计划 / Research Proposal）+ 已发实验工作**上。所以每解读一块老师的方向，都要紧跟一句「这一块怎么和你的 RP 对上」：
    - 具体到 RP 里的**机制 / 方法 / 体系**（如多键网络、VFc 微凝胶、CMCS 水凝胶、辐照剂量响应、逆向设计），不要泛泛说「都做软物质」。
    - 给学生**一句可以直接说出口的话**：「我在 RP 里做的 X，正好能用你的 Y 方法 / 延伸到你的 Z 问题」。这是 call 里把「我了解你」升级成「我们能一起做什么」的关键。
    - 准备 §四「问老师的问题」时，问题要从 RP 的真实接口长出来（你的方法能不能用在我 RP 的这个环节），不是通用礼貌问题。
    - **必须先读学生的 RP 原文**（vault 里 `个性化材料` 下的 RP / 研究计划文档）再写，不能凭印象编 RP 内容。RP 写了什么就接什么，没写的别替学生承诺。

15. **🚨 绝不暴露我们这边的失误 / 邮件出过错。** 学生不一定读过、更不一定理解我们替他写的套磁信。如果发现信里把某篇论文安错了人、或写错了，**准备包里直接用正确信息呈现就好，绝不点破「我们之前发的信写错了」**。不要出现「邮件里误把 X 当成你的论文」「之前发的版本有误」这类话——学生看到会不安、对我们失去信任。正确做法：把老师**真实**的代表作自然地讲清楚、接到 RP 上，当作我们一直就准备得很扎实。万一学生主动问起信里某篇，话术是「那篇相关方向我也看了，更核心的是他这几篇」，平滑带过，不自我归罪。

## 自查清单（交付前过一遍）
- [ ] grep `—` / `——` / `–` = 0
- [ ] grep `不是.*而是` / `不只是.*更是` / `与其说` = 0
- [ ] 引号是否都在引真实原话 / 标题？装饰性引号删掉
- [ ] 有没有成语化修辞 / 夸张词 / 水词过渡
- [ ] 论文断言可追溯，无臆造；每篇都 CrossRef 核过作者归属（老师本人在署名里）
- [ ] 每块老师方向都接回了学生 RP（具体机制 / 方法，不泛泛），且 RP 内容来自真实原文
- [ ] 全文无「我们之前发的信写错 / 安错论文」之类自我归罪、不暴露我方失误
- [ ] 三种格式都用上了（正文 / `>` 引用块 / `[!INSIGHT]` callout）
- [ ] 任何两个 `>` 块之间都有正文 / 列表 / `---` 隔开（只隔空行或只隔标题 = 改；脚本：相邻 `>` 组之间无正文/列表/`---` 就 flag）
- [ ] 结尾有 APA7 ref list
